Why FaultRay?

Three pillars that make infrastructure resilience testing safe, fast, and provable.

Zero Risk

Simulate entirely in-memory. No agents installed on your servers, no production access required, no chance of real outages. Pure mathematical simulation that runs on your laptop.

2,000+ Scenarios

Auto-generated from your topology YAML. Single-component failures, cascading outages, traffic spikes, network partitions, and multi-region disasters -- all covered automatically.

Mathematical Proof

The N-Layer Availability Model proves your SLA is achievable -- or identifies exactly why it is not. Separate Software, Hardware, and Theoretical limits to find your true ceiling.

How It Works

From zero to resilience proof in three steps

1

Define Your Infrastructure

Describe your topology in a simple YAML file. Services, dependencies, replicas, and regions.

# infra.yaml
topology:
  name: my-platform
  services:
    - name: api-gateway
      replicas: 3
      dependencies: [auth, db]
2

Run Simulation

A single command triggers 2,000+ chaos scenarios against your topology. Takes seconds, not hours.

$ faultray simulate \
    --topology infra.yaml \
    --scenarios all
Running 2,147 scenarios...
Done in 6.2s
3

Get Results

Resilience score, SPOF analysis, cascade maps, compliance reports, and financial risk quantification.

$ faultray report
Availability: 99.991% (3.95 nines)
SPOFs found: 2
Cascade risk: LOW
Report saved: report.html

Getting Started

Up and running in under a minute

Install 
$ pip install faultray
infra.yaml 
topology:
  name: my-saas-platform
  regions:
    - name: us-east-1
      zones: [a, b, c]
  services:
    - name: api-gateway
      replicas: 3
      dependencies: [auth-service, database]
    - name: auth-service
      replicas: 2
      dependencies: [database, cache]
    - name: database
      type: rds-multi-az
      replicas: 2
    - name: cache
      type: elasticache
      replicas: 3
Run simulation 
$ faultray simulate --topology infra.yaml --scenarios all
Running 2,147 scenarios across 5 engines...
Completed in 6.2s | Pass: 2,142 | Fail: 5
Availability Ceiling: 99.991% (3.95 nines)

$ faultray report --format html --output report.html
Report saved: report.html
