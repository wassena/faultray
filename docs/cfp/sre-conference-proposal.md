# CFP提案書 — SRE系カンファレンス用

## 提案書A: メインセッション（30分）

### タイトル
**本番を壊さずに2,000シナリオの障害シミュレーション — 数学的Chaos Engineeringの設計と実装**

### カテゴリ
Tech / Architecture

### 対象者
中級〜上級（SRE/インフラエンジニア）

### 概要（400字）

Chaos Engineeringは「本番に障害を注入して耐性を検証する」手法として定着しました。しかし金融機関や医療システムでは、本番への障害注入が許可されないケースが多く存在します。

本セッションでは、本番環境に一切触れずにインフラの可用性限界を数学的に証明するアプローチを紹介します。個人開発のOSSツール「FaultRay」を題材に、以下を解説します:

1. **グラフ理論によるインフラモデリング** — NetworkXで依存関係をDAGとして表現し、2,000以上の障害シナリオをインメモリで実行
2. **3層可用性限界モデル** — ソフトウェア限界(4.00 nines)→ハードウェア限界(5.91 nines)→理論限界(6.65 nines)を物理法則から導出
3. **AIエージェント耐性テスト** — MCPサーバー経由でAIエージェントの障害伝播を解析
4. **DORA規制対応** — EU金融規制が求めるレジリエンステストのエビデンスを自動生成

テスト32,000件・USPTO仮特許出願済みの実装から得た知見を、設計判断の理由とともに共有します。

### 聴衆が持ち帰れるもの
- 本番障害注入なしでレジリエンスを定量評価する具体的手法
- グラフ理論を使った障害伝播シミュレーションの実装パターン
- 可用性の「理論的上限」を数学的に算出する考え方
- DORA規制が求めるテストエビデンスの具体例

### スピーカー紹介
前田雄太郎。SREエンジニア。インフラの可用性限界を数学的に証明するOSSツール「FaultRay」を個人開発。USPTO仮特許出願済み（US 64/010,200）。Zenodoに論文公開（DOI: 10.5281/zenodo.19139911）。

## 提案書B: 代替案（AI×SREトレンド寄せ）

### タイトル
**AIエージェント時代のSRE — LLMが落ちたらサービスはどうなるか？を1人で32,000テスト書いて検証した話**

### カテゴリ
Future / Tech

### 概要（400字）

LLMベースのAIエージェントがプロダクションに入り始めた今、新しい障害モードが生まれています。「LLMプロバイダーのレート制限」「トークン上限によるコンテキスト喪失」「ハルシネーションの伝播」— これらは従来のChaos Engineeringでは検証できません。

本セッションでは、AIエージェントを含むシステムの障害シミュレーションに取り組んだ経験を共有します。個人開発のOSSツール「FaultRay」にAIエージェント耐性テスト機能を実装する過程で得た知見:

1. **AIエージェントの障害分類** — LLM固有の障害モード(レート制限、コンテキスト溢れ、ハルシネーション伝播)をどう体系化したか
2. **MCP経由の障害注入** — 12ツールのMCPサーバーでAIエージェントのレジリエンスを非破壊テスト
3. **従来インフラとの障害カスケード** — AIエージェントの障害がDBやキャッシュに波及するパターン
4. **1人開発で32,000テスト** — AIを活用した開発プロセスと品質管理の実践

「AIが壊れたとき、あなたのサービスは何秒で復旧できますか？」に答えられるようになるセッションです。

### 聴衆が持ち帰れるもの
- AIエージェント固有の障害モード分類と対策
- MCPを使った非破壊レジリエンステストの実装方法
- 1人開発でも品質を担保するテスト戦略（32,000テストの体系）
- AI時代のSREが準備すべきこと
